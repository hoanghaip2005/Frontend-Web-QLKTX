# PM Docs Snapshot

Folder này là bản copy local của các PM/technical docs cần cho web frontend. Members làm trong `Frontend-Web-QLKTX` đọc các file ở đây thay vì clone hoặc mở repo `D:\NGONGOCDANGKHOA`.

## Files

- `project-overview.md`: MVP boundary, user groups, risks.
- `product-backlog.md`: user stories, priority, sprint target.
- `prototype-spec.md`: screen list, flow, sample data, acceptance criteria.
- `definition-of-done.md`: quality bar và evidence checklist.
- `release-plan-fixed-date.md`: 4 sprint MVP scope guardrail.

## Refresh Rule

Member 1/PM refresh các file này từ `D:\NGONGOCDANGKHOA\docs` khi upstream scope thay đổi. Sau khi refresh, kiểm tra lại `docs/frontend-architecture.md`, `docs/team-work-rules.md`, `README.md`, và `AGENTS.md` để tránh lệch rule.
